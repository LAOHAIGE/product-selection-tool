# 规则配置中心设计

## 目标

用户可以在工具页面内编辑 ASIN 筛选阈值、ASIN AI 总结 Prompt 和竞品分析 Prompt。配置保存在当前浏览器，并随工作区继续使用。

## 界面

- 将现有 AI 配置区扩展为“规则配置中心”，包含 `ASIN 详情规则`、`AI 分析规则`、`竞品分析规则` 三个标签页。
- ASIN 详情使用带单位的数字输入框，支持月销量、销量增长率、价格、评分、FBA 费用、FBA/价格比、毛利率、Review 数、卖家数、上架天数、强候选分、观察候选分和高风险分。
- 百分比字段在页面按百分数编辑，传入分析引擎时转换为 0-1 小数。
- AI 和竞品规则继续使用多行 Prompt 编辑器。DeepSeek Key 仍与规则一起显示，但不写入工作区数据。
- 提供“保存规则”和“恢复默认”。恢复默认只填充表单，需再点击保存才生效。

## 数据与行为

- 浏览器配置增加 `selectionRules`，与现有 `prompt`、`competitorPrompt` 一起保存。旧配置缺失字段时自动补默认值。
- 导入产品表时将当前 `selectionRules` 传给 `/api/analyze`，后端校验后用于本次分析。
- 保存数字规则后调用 `POST /api/reanalyze`，对当前候选池重新计算分数、详情结论和候选状态。
- 重新分析以 ASIN 合并结果，必须保留 `reviewed`、`reviewedAt`、`sif`、`competitors` 和 `aiSummary` 等已补数据。
- Prompt 保存后只影响下一次 AI 调用，不自动批量重新调用 DeepSeek。
- 保存成功后更新 IndexedDB 工作区；Render 重启后由现有恢复机制同步回服务器。

## 校验与错误

- 所有数值必须是有限数；价格、销量、费用、Review、卖家数和天数不得为负数。
- 评分限制为 0-5，百分比限制为 0%-100%，分数阈值限制为 0-100。
- 强候选机会分不得低于观察候选机会分。
- 页面和后端都执行校验。失败时保留原配置和当前候选池，并显示具体字段错误。

## 测试

- 规则默认值、浏览器保存、旧配置兼容和恢复默认。
- 有效/无效数值、百分比转换和分数阈值关系。
- 产品导入使用自定义规则。
- 当前候选池重新分析后状态变化，但已看、关键词、竞品和 AI 结果保留。
- Prompt 保存后的下一次 AI 请求使用新规则，且不会触发批量 AI 请求。
